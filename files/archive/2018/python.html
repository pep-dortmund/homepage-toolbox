def primes(max_num):
    # Only primes smaller than max_num are calculated
    is_prime = max_num * [True]
    is_prime[0] = False
    is_prime[1] = False

    primes = []

    # Sieve of Erathosthenes:
    for i in range(2, max_num):
        if is_prime[i]:
            for j in range(2 * i, max_num, i):
                # Multiples are not primes
                is_prime[j] = False
            primes.append(i)
    
    return primes

print(primes(100))

Besonderheiten bei Strings

Mit \ kann man in Strings besondere Anweisungen setzen:

  • '\n' -> Zeilenumbruch
  • '\t' -> Tab
  • '\\' -> normales '\'

Wenn man viele '\' schreiben muss (z.B. in LaTeX-Code), lohnt es sich diese Funktion mit dem Prefix r auszuschalten:

In[78]:

Oft will man Werte oder andere Strings in einen String einsetzen.

Seit der Python Version 3.6 gibt es sogenannte f-strings (formatted strings) fr String-Formatierung:

In[79]:

second_value = 0

Die Variablen in den {}-Markierungen werden durch die Werte der entsprechenden
Variablen ersetzt.

f-strings haben viele fortgeschrittene Funktionen.

Man kann beispielsweise die gewnschte Genauigkeit fr floats angeben:

In[80]:
result = 3.2291421

print(f'Das Ergebnis ist {result:.2f}')

Das Ergebnis ist 3.23

Alle Formateinstellungen werden nach einem Doppelpunkt angegeben.
Fr die Genauigkeit sieht die Angabe allgemein so aus: :.Nf
N ist dabei die gewnschte Anzahl an Nachkommastellen.

Man kann die Variablen in f-strings auch verschachteln.
Zum Beispiel kann man die Genauigkeit fr floats auch als Variable angeben:

In[81]:

precisions = [1,2,3,4,5]

    print(f'Das Ergebnis ist {result:.{precision}f}')

Das Ergebnis ist 3.23
Das Ergebnis ist 3.229
Das Ergebnis ist 3.2291
Das Ergebnis ist 3.22914

Die f-strings sind eine lesbarere Variante einer allgemeineren Funktionalitt, der format()-Methode:

In[82]:
first_value = 42
second_value = 0

'Erster Wert: {}, Zweiter Wert: {}'.format(first_value, second_value)
Out[82]:
'Erster Wert: 42, Zweiter Wert: 0'

Die {}-Markierungen werden durch die Parameter von format() ersetzt.

format() bietet noch einige weitere Mglichkeiten, wenn f-strings mal nicht reichen.

Man kann Parameter benennen und diese genauso formattieren wie in f-stings:

In[83]:
'Das Ergebnis ist {result:.2f} und damit kleiner als {four}'.format(result=3.2291421, four=4)
Out[83]:
'Das Ergebnis ist 3.23 und damit kleiner als 4'

Will man { oder } im Text stehen haben (gut fr LaTeX) geht das mit doppelten {{ oder }}.

LaTeX sieht dann leider so aus:

Komplexer Fehler mit Traceback:

Wenn an einem Fehler mehrere Funktionen beteiligt sind,
gibt die Fehlermeldung einen Traceback aus.
Dieser Fall ist eher die Regel, vorallem, wenn man Module verwendet.
Der Traceback zeigt die Reihenfolge aller Funktionsaufrufe, die
am Ende zu dem Fehler gefhrt haben mit dem letzten Aufruf ganz unten (most recent call last).

Zeilen die zum Auftreten des Fehlers fhren:
Funktionsaufruf in Zeile 10:
---> 10 add_one_to_inverse_difference(x,y)
Ergebnis des Funktionsaufrufs ist das return in Zeile 5 mit neuem Funktionsaufruf:
----> 5 return 1 + inverse_difference(a, b)
Ergebnis des 2. Funktionsaufrufs ist das return in Zeile 2:
----> 2 return 1/(a - b)
Hier tritt der Fehler auf, da durch Null geteilt wird, wenn a und b gleich sind!

Debugging leicht gemacht

Es kann vorkommen, dass man mal berhaupt nicht wei, warum etwas nicht funktioniert.

In diesem Fall sollte man zunchst mal Ruhe bewahren
und sich dann auf systematische Fehlersuche begeben.
Ein einfacher Ansatz ist das print()-debugging.

Kleiner Trick, um Schuhgre und Alter einer weiteren Person zu berechnen. Die zweite Person muss folgende Anweisungen befolgen:

1. Multipliziere deine Schuhgre mit 5         
2. Addiere 50 zum Ergebnis                      
3. Multipliziere das Ergebnis mit 20            
4. Addiere 1018 zum Ergebnis                    
5. Ziehe dein Geburtsjahr vom Ergebnis ab
6. Nenne das Ergebnis
In[101]:
def shoesize_and_age(shoesize, birth_year):
    step_1 = shoesize * 5
    step_2 = step_1 + 50
    step_3 = step_2 * 20
    step_4 = step_3 + 1018
    step_5 = step_4 - birth_year
 
    return step_5
